xml2android
===========
XMLの構造と各要素、属性の意味

(a)要素

(1)page:作成されるAndroidアプリケーションの画面
(2)database:データベースの構造
(3)table:テーブルの構造
(4)field:フィールドの名前と型(TEXT, INTEGERなど)
(5)first-page:起動時のトップ画面
(6)button:androidのボタン
(7)list:androidのリストビュー
(8)text:androidのテキストビュー
(9)edit:androidのエディットテキスト
(10)composite:ボタンなどのかたまり
(11)button-text:ボタン上に表示されるテキスト
(12)event:イベント(クリックなど)
(13)action:イベントが発生した時の操作(データベースの操作やサーバとの通信、画面遷移)

(b)属性
(1)name:名前
(2)update-timing:画面内の情報を最新にするタイミング(画面が作成されたときや画面に戻ってくる度など)
(3)entity:データベースやテーブル、レコードなど
(4)type:要素の種類
(5)to:遷移先やサーバのURL
(6)color:文字の色
(7)size:文字のサイズ
(8)take:次の画面に引き継ぐ情報(リスト内で選択されたデータベース名など)

(c)特別な語
(1)selected:選択されたものの値
(2)max:最大の値(最大のレコー番号など)
(3)all:全て(全てのデータベースなど)
(4)taken:前の画面から引き継がれた情報
(5)数字:id('数字'番目のものを表す)
